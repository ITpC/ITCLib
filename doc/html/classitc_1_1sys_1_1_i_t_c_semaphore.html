Detailed Description

Simulates a semaphore using a Mutex. Its differ to the RawPosixSemaphore or PosixSemaphore defined for platforms other than Linux and Solaris in performance/safety tradeoff behavior. The favor given to the safety(However bugs are sommewhere here ...). No undefined behavior by destroying. No possibility to use uninitialized semaphore. ITCSemaphore should be absolutelly safe (if no bugs in this realisation are present) usleep used here to suspend threads execution on wait and on destroy. Solution is unportable, unperformant and bugy as well.

Exceptions
ITCException(Can_not_init_semaphore,EINVAL)
ITCException(Can_not_get_semaphore_value,EINVAL)
ITCException(Can_not_wait_on_semaphore,EINVAL)
Propagateall Mutex exceptions further if any.
